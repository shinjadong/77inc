# 모델 개요

> Claude는 Anthropic에서 개발한 최첨단 대규모 언어 모델 제품군입니다. 이 가이드는 당사의 모델을 소개하고 성능을 비교합니다.

---

## 모델 선택하기

어떤 모델을 사용할지 확실하지 않다면, **Claude Sonnet 4.5**부터 시작하는 것을 권장합니다. 대부분의 사용 사례에서 지능, 속도, 비용의 최적 균형을 제공하며, 코딩 및 에이전트 작업에서 뛰어난 성능을 보입니다.

현재의 모든 Claude 모델은 다음을 지원합니다:
- 텍스트 및 이미지 입력
- 텍스트 출력
- 다국어 기능
- 비전 처리

모델은 다음을 통해 사용할 수 있습니다:
- Anthropic API
- AWS Bedrock
- Google Vertex AI

모델을 선택한 후, [첫 번째 API 호출을 하는 방법을 알아보세요](/ko/docs/get-started).

### API 별칭 참고사항

별칭은 자동으로 가장 최신 모델 스냅샷을 가리킵니다. 새로운 모델 스냅샷을 출시할 때, 우리는 일반적으로 새 출시 후 1주일 이내에 별칭을 모델의 최신 버전으로 마이그레이션합니다.

**권장사항:**
- **실험**: 별칭 사용 (예: `claude-sonnet-4-5`)
- **프로덕션**: 특정 모델 버전 사용 (예: `claude-sonnet-4-5-20250929`) - 일관된 동작을 보장합니다.

### 가격 세부정보

배치 API 할인, 프롬프트 캐싱 요금, 확장 사고 비용 및 비전 처리 수수료를 포함한 완전한 가격 정보는 [가격 페이지](/ko/docs/about-claude/pricing)를 참조하세요.

### 컨텍스트 윈도우

**Claude Sonnet 4.5**는 다음을 지원합니다:
- **기본**: 200K 토큰 (~150K 단어, ~680K 유니코드 문자)
- **베타**: 1M 토큰 (~750K 단어, ~3.4M 유니코드 문자)

1M 토큰 컨텍스트를 사용하려면 API 요청에 `context-1m-2025-08-07` 베타 헤더를 포함하세요. [장문 컨텍스트 가격](/ko/docs/build-with-claude/context-windows#1m-token-context-window)은 200K 토큰을 초과하는 요청에 적용됩니다.

### 지식 기준일 설명

- **신뢰할 수 있는 지식 기준일**: 모델의 지식이 가장 광범위하고 신뢰할 수 있는 날짜를 나타냅니다.
- **학습 데이터 기준일**: 사용된 학습 데이터의 더 넓은 날짜 범위입니다.

**예시**: Claude Sonnet 4.5는 2025년 7월까지 공개적으로 이용 가능한 정보로 학습되었지만, 그 지식은 2025년 1월까지 가장 광범위하고 신뢰할 수 있습니다. 자세한 내용은 [Anthropic의 투명성 허브](https://www.anthropic.com/transparency)를 참조하세요.

### 모델 스냅샷 일관성

동일한 스냅샷 날짜(예: 20240620)를 가진 모델은 모든 플랫폼에서 동일하며 변경되지 않습니다. 모델 이름의 스냅샷 날짜는 일관성을 보장하고 개발자가 다양한 환경에서 안정적인 성능에 의존할 수 있도록 합니다.

### AWS Bedrock 및 Google Vertex AI 엔드포인트

**Claude Sonnet 4.5** 이후 모든 모델은 AWS Bedrock 및 Google Vertex AI에서 두 가지 엔드포인트 유형을 제공합니다:

- **글로벌 엔드포인트**: 최대 가용성을 위한 동적 라우팅
- **지역 엔드포인트**: 특정 지리적 영역을 통한 보장된 데이터 라우팅

자세한 내용은 [타사 플랫폼 가격 섹션](/ko/docs/about-claude/pricing#third-party-platform-pricing)을 참조하세요.

### Claude Sonnet 3.7 최대 출력 확장

Claude Sonnet 3.7의 최대 출력 토큰 길이를 128K 토큰으로 증가시키려면 API 요청에 베타 헤더 `output-128k-2025-02-19`를 포함하세요.

**권장사항**: 더 긴 출력을 생성할 때 타임아웃을 피하기 위해 [스트리밍 Messages API](/ko/docs/build-with-claude/streaming)를 사용하는 것을 강력히 권장합니다. 자세한 내용은 [장문 요청](/ko/api/errors#long-requests)에 대한 지침을 참조하세요.

## 프롬프트 및 출력 성능

Claude 4 모델은 다음 분야에서 뛰어납니다:

### 성능 영역

- **추론**: 복잡한 논리 문제 해결
- **코딩**: 고품질 코드 생성 및 디버깅
- **다국어**: 40개 이상의 언어 지원
- **장문 컨텍스트**: 200K 토큰 이상 처리
- **정직성**: 명확한 제한 사항 인식 및 불확실성 표현
- **이미지 처리**: 고급 비전 및 이미지 분석

자세한 내용은 [Claude 4 블로그 게시물](http://www.anthropic.com/news/claude-4)을 참조하세요.

### 매력적인 응답

Claude 모델은 풍부하고 인간다운 상호작용이 필요한 애플리케이션에 이상적입니다.
**더 간결한 응답이 필요한 경우:**
프롬프트를 조정하여 모델이 원하는 출력 길이로 향하도록 할 수 있습니다. 자세한 내용은 [프롬프트 엔지니어링 가이드](/ko/docs/build-with-claude/prompt-engineering)를 참조하세요.

**Claude 4 프롬프팅 모범 사례:**
[Claude 4 모범 사례 가이드](/ko/docs/build-with-claude/prompt-engineering/claude-4-best-practices)를 참조하세요.

### 출력 품질

이전 모델 세대에서 Claude 4로 마이그레이션할 때, 전반적인 성능에서 더 큰 개선을 알 수 있습니다.
