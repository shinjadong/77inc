# 프롬프트 개선기를 사용하여 프롬프트 최적화하기

> 프롬프트 개선기를 사용하여 프롬프트를 최적화하는 방법을 알아보세요

<Note>
프롬프트 개선기는 확장된 사고 기능을 포함한 모든 Claude 모델과 호환됩니다. 확장된 사고 모델에 특화된 프롬프팅 팁은 [여기](/ko/docs/build-with-claude/extended-thinking)를 참조하세요.
</Note>

프롬프트 개선기는 자동화된 분석과 향상을 통해 프롬프트를 빠르게 반복하고 개선하는 데 도움을 줍니다. 높은 정확도가 필요한 복잡한 작업에 대해 프롬프트를 더욱 견고하게 만드는 데 뛰어납니다.

## 시작하기 전에

다음이 필요합니다:

* 개선할 [프롬프트 템플릿](/ko/docs/build-with-claude/prompt-engineering/prompt-templates-and-variables)
* Claude의 현재 출력에 대한 피드백 (선택사항이지만 권장)
* 예시 입력과 이상적인 출력 (선택사항이지만 권장)

## 프롬프트 개선기 작동 방식

프롬프트 개선기는 4단계로 프롬프트를 향상시킵니다:

1. **예시 식별**: 프롬프트 템플릿에서 예시를 찾아 추출합니다
2. **초기 초안**: 명확한 섹션과 XML 태그가 있는 구조화된 템플릿을 생성합니다
3. **사고 과정 개선**: 상세한 추론 지침을 추가하고 개선합니다
4. **예시 향상**: 새로운 추론 과정을 보여주도록 예시를 업데이트합니다

개선 모달에서 이러한 단계들이 실시간으로 진행되는 것을 볼 수 있습니다.

## 얻을 수 있는 것

프롬프트 개선기는 다음과 같은 템플릿을 생성합니다:

* Claude의 추론 과정을 안내하고 일반적으로 성능을 향상시키는 **상세한 사고 과정 지침**
* 다양한 구성 요소를 분리하기 위해 **XML 태그를 사용한 명확한 구성**
* 입력에서 출력까지 **단계별 추론을 보여주는 표준화된 예시 형식**
* Claude의 초기 응답을 안내하는 **전략적 미리 채우기**

<Note>
예시들이 Workbench UI에서는 별도로 나타나지만, 실제 API 호출에서는 첫 번째 사용자 메시지의 시작 부분에 포함됩니다. "**\</> Get Code**"를 클릭하여 원시 형식을 보거나 Examples 박스를 통해 예시를 원시 텍스트로 삽입하세요.
</Note>

## 프롬프트 개선기 사용 방법

1. **프롬프트 템플릿 제출**: 개선하고자 하는 기본 프롬프트를 입력합니다
2. **피드백 추가**: Claude의 현재 출력에 대한 문제점에 대한 피드백을 추가합니다  
   (예: "요약이 전문가 대상에게는 너무 기본적입니다")
3. **예시 포함**: 예시 입력과 이상적인 출력을 포함합니다
4. **검토 및 적용**: 개선된 프롬프트를 검토하고 적용합니다

## 테스트 예시 생성

아직 예시가 없나요? 저희 [테스트 케이스 생성기](/ko/docs/test-and-evaluate/eval-tool#creating-test-cases)를 사용하여:

1. **샘플 입력 생성**: 다양한 상황을 대표하는 입력 데이터를 만듭니다
2. **Claude 응답 수집**: Claude의 초기 응답을 받습니다
3. **응답 편집**: 이상적인 출력에 맞게 응답을 수정합니다
4. **예시 추가**: 다듬어진 예시를 프롬프트에 추가합니다

## 프롬프트 개선기를 사용해야 하는 경우

프롬프트 개선기는 다음과 같은 경우에 가장 효과적입니다:

* **복잡한 작업**: 상세한 추론이 필요한 복잡한 작업을 수행할 때
* **정확도 중심**: 속도보다 정확도가 더 중요한 상황에서
* **품질 개선 필요**: Claude의 현재 출력에 상당한 개선이 필요한 문제를 해결할 때

<Note>
지연 시간이나 비용에 민감한 애플리케이션의 경우, 더 간단한 프롬프트 사용을 고려하세요. 프롬프트 개선기는 더 길고 철저하지만 느린 응답을 생성하는 템플릿을 만듭니다.
</Note>

## 개선 예시

다음은 프롬프트 개선기가 기본 분류 프롬프트를 어떻게 향상시키는지 보여줍니다:

<Accordion title="원본 프롬프트">
```text
다음 Wikipedia 기사 제목 목록에서 이 문장이 어느 기사에서 나온 것인지 식별하세요.
기사 제목만 응답하고 다른 것은 아무것도 하지 마세요.
기사 제목:
{{titles}}
분류할 문장:
{{sentence}}
```
</Accordion>

<Accordion title="개선된 프롬프트">
```text
당신은 문장을 Wikipedia 기사 제목에 매칭하는 데 특화된 지능형 텍스트 분류 시스템입니다. 
당신의 작업은 제공된 기사 제목 목록을 기반으로 주어진 문장이 어느 Wikipedia 기사에 속할 가능성이 가장 높은지 식별하는 것입니다.
먼저 다음 Wikipedia 기사 제목 목록을 검토하세요:
<article_titles>
{{titles}}
</article_titles>
이제 분류해야 할 이 문장을 고려하세요:
<sentence_to_classify>
{{sentence}}
</sentence_to_classify>
당신의 목표는 제공된 목록에서 주어진 문장과 가장 잘 일치하는 기사 제목을 결정하는 것입니다. 
다음 단계를 따르세요:
1. 문장에서 핵심 개념들을 나열하세요
2. 각 핵심 개념을 기사 제목들과 비교하세요
3. 가장 관련성이 높은 상위 3개 제목을 순위를 매기고 왜 관련성이 있는지 설명하세요
4. 문장의 내용을 가장 잘 포괄하거나 관련된 가장 적절한 기사 제목을 선택하세요
<analysis> 태그로 분석을 감싸세요. 다음을 포함하세요:
- 문장에서 추출한 핵심 개념 목록
- 각 핵심 개념과 기사 제목들의 비교
- 설명과 함께 가장 관련성이 높은 상위 3개 제목의 순위
- 최종 선택과 추론
분석 후, 최종 답변을 제공하세요: 목록에서 가장 적절한 단일 Wikipedia 기사 제목.
선택한 기사 제목만 출력하고, 추가 텍스트나 설명은 하지 마세요.
```
</Accordion>

### 개선된 프롬프트의 장점
개선된 프롬프트가 어떻게 다음과 같은 요소들을 추가하는지 주목하세요:
* **명확한 단계별 추론 지침**: 체계적인 분석 과정 제시
* **XML 태그를 사용한 구성**: 입력과 출력을 명확히 분리
* **명시적인 출력 형식 요구사항**: 정확한 응답 형식 지정
* **Claude를 분석 과정을 통해 안내**: 논리적 사고 과정 유도

## 문제 해결

일반적인 문제와 해결책:

### 예시가 출력에 나타나지 않음
**문제**: 프롬프트에 포함한 예시가 실제 응답에서 보이지 않습니다.
**해결책**: 예시가 XML 태그로 적절히 형식화되어 있고 첫 번째 사용자 메시지의 시작 부분에 나타나는지 확인하세요.

### 사고 과정이 너무 장황함
**문제**: Claude의 추론 과정이 너무 길어서 실용적이지 않습니다.
**해결책**: 원하는 출력 길이와 세부 수준에 대한 구체적인 지침을 추가하세요.

### 추론 단계가 필요에 맞지 않음
**문제**: 생성된 추론 단계가 특정 사용 사례와 맞지 않습니다.
**해결책**: 특정 사용 사례에 맞게 단계 섹션을 수정하고 커스터마이징하세요.
