# 예시(멀티샷 프롬프팅)를 사용하여 Claude의 행동 유도하기

<Note>
  While these tips apply broadly to all Claude models, you can find prompting tips specific to extended thinking models [here](/en/docs/build-with-claude/prompt-engineering/extended-thinking-tips).
</Note>

예시는 Claude가 필요한 것을 정확하게 생성하도록 하는 비밀 무기와 같은 지름길입니다. 프롬프트에 잘 만들어진 예시 몇 개를 제공함으로써 Claude 출력의 정확성, 일관성 및 품질을 크게 향상시킬 수 있습니다.
퓨샷 또는 멀티샷 프롬프팅이라고 알려진 이 기술은 특히 구조화된 출력이나 특정 형식을 준수해야 하는 작업에 효과적입니다.

<Tip>**프롬프트 강화하기**: Claude에게 원하는 것을 정확히 보여주기 위해 3-5개의 다양하고 관련된 예시를 포함하세요. 예시가 많을수록 특히 복잡한 작업에서 더 나은 성능을 보입니다.</Tip>

## 왜 예시를 사용해야 하나요?

* **정확성**: 예시는 지침의 잘못된 해석을 줄입니다.
* **일관성**: 예시는 균일한 구조와 스타일을 강제합니다.
* **성능**: 잘 선택된 예시는 복잡한 작업을 처리하는 Claude의 능력을 향상시킵니다.

## 효과적인 예시 만들기

최대한의 효과를 위해 예시가 다음과 같은지 확인하세요:

* **관련성**: 예시가 실제 사용 사례를 반영합니다.
* **다양성**: 예시가 엣지 케이스와 잠재적 과제를 다루며, Claude가 의도하지 않은 패턴을 우연히 습득하지 않도록 충분히 다양합니다.
* **명확성**: 예시가 구조를 위해 `<example>` 태그로 감싸져 있습니다(여러 개인 경우 `<examples>` 태그 안에 중첩).

<Tip>Claude에게 예시의 관련성, 다양성 또는 명확성을 평가하도록 요청하세요. 또는 Claude가 초기 세트를 기반으로 더 많은 예시를 생성하도록 하세요.</Tip>
